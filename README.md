**Требования к выполнению работы:**

    Тестовое покрытие должно быть сформировано на основании набора прецедентов использования сайта.
    Тестирование должно осуществляться автоматически - с помощью системы автоматизированного тестирования Selenium.

# Описание набора тестовых сценариев

## Группа 1: Авторизация и регистрация

### Сценарий: TS-01-01 Регистрация с невалидным email
**Дано:** Пользователь на форме регистрации  
**Когда:** Вводит email в некорректном формате (например, «not-an-email»)  
**Тогда:** Отображается сообщение об ошибке email, регистрация не выполняется

### Сценарий: TS-01-02 Успешный вход в аккаунт
**Дано:** Пользователь на странице входа  
**Когда:** Вводит корректный логин и пароль, нажимает «Войти»  
**Тогда:** Открыт личный кабинет (URL: https://hosting.timeweb.ru/), форма авторизации скрыта

### Сценарий: TS-01-03 Вход с неверным паролем
**Дано:** Пользователь на странице входа  
**Когда:** Вводит валидный email и неверный пароль  
**Тогда:** Отображается сообщение об ошибке («Неверный логин или пароль»)

### Сценарий: TS-01-04 Вход с пустым паролем
**Дано:** Пользователь на странице входа  
**Когда:** Вводит email, оставляет пароль пустым и нажимает «Войти»  
**Тогда:** Отображается сообщение «Необходимо заполнить «Пароль»»

---

## Группа 2: Домены

### Сценарий: TS-02-01 Проверка свободного домена и переход к оформлению
**Дано:** Пользователь авторизован и находится на странице регистрации доменов  
**Когда:** Вводит свободное доменное имя, дожидается проверки доступности и нажимает «Зарегистрировать»  
**Тогда:** Кнопка регистрации активна, происходит переход на страницу оформления заказа (URL содержит #order/)

### Сценарий: TS-02-01 Проверка занятого домена
**Дано:** Пользователь авторизован и находится на странице регистрации доменов  
**Когда:** Вводит уже зарегистрированный домен  
**Тогда:** Отображается сообщение «Домен уже зарегистрирован. Выберите другой домен.», кнопка регистрации неактивна

---

## Группа 3: Техническая поддержка

### Сценарий: TS-03-01 Создание заявки в поддержку
**Дано:** Пользователь в форме создания обращения  
**Когда:** Вводит текст сообщения и нажимает «Отправить сообщение»  
**Тогда:** Тикет создан, происходит переход на страницу /tickets со списком сообщений

### Сценарий: TS-03-02 Заявка с вложением файла
**Дано:** Пользователь в форме создания обращения  
**Когда:** Вводит текст, прикрепляет файл и отправляет  
**Тогда:** Тикет создан, вложение принято, выполнен переход на /tickets

### Сценарий: TS-03-03 Отправка заявки без описания
**Дано:** Пользователь в форме создания обращения  
**Когда:** Оставляет поле сообщения пустым и нажимает «Отправить сообщение»  
**Тогда:** Отображается сообщение «Введите текст сообщения перед отправкой», отправка блокируется

### Сценарий: TS-03-04 Заявка с длинным описанием
**Дано:** Пользователь в форме создания обращения  
**Когда:** Вводит текст длиной ~5000 символов и отправляет  
**Тогда:** Тикет успешно создаётся (переход на /tickets), либо срабатывает валидация без критических ошибок
