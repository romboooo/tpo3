**Требования к выполнению работы:**

    Тестовое покрытие должно быть сформировано на основании набора прецедентов использования сайта.
    Тестирование должно осуществляться автоматически - с помощью системы автоматизированного тестирования Selenium.
    Шаблоны тестов должны формироваться при помощи Selenium IDE и исполняться при помощи Selenium RC в браузерах Firefox и Chrome.
    Предполагается, что тестируемый сайт использует динамическую генерацию элементов на странице, т.е. выбор элемента в DOM должен осуществляться не на основании его ID, а с помощью XPath.

# Описание набора тестовых сценариев

## Группа 1: Авторизация и регистрация

### Сценарий: TS-01-01 Успешная регистрация
**Дано:** Пользователь на главной странице  
**Когда:** Вводит ФИО, валидный email и пароль, нажимает «Зарегистрироваться»  
**Тогда:** Аккаунт создан, открыт личный кабинет

### Сценарий: TS-01-02 Регистрация с занятым email
**Дано:** Пользователь на форме регистрации  
**Когда:** Вводит уже зарегистрированный email  
**Тогда:** Отображается сообщение «Email уже зарегистрирован»

### Сценарий: TS-01-03 Регистрация с невалидным email
**Дано:** Пользователь на форме регистрации  
**Когда:** Вводит email в некорректном формате (например, «test@»)  
**Тогда:** Отображается сообщение «Некорректный формат email»

### Сценарий: TS-01-04 Успешный вход в аккаунт
**Дано:** Пользователь на главной странице  
**Когда:** Вводит логин и пароль, нажимает «Войти»  
**Тогда:** Открыт личный кабинет, отображается имя пользователя

### Сценарий: TS-01-05 Вход с неверным паролем
**Дано:** Пользователь на форме авторизации  
**Когда:** Вводит валидный email и неверный пароль  
**Тогда:** Отображается сообщение «Неверный логин или пароль»

### Сценарий: TS-01-06 Вход с пустым паролем
**Дано:** Пользователь на форме авторизации  
**Когда:** Вводит email, оставляет пароль пустым  
**Тогда:** Отображается сообщение «Введите пароль»

## Группа 2: Хостинг и тарифы

### Сценарий: TS-02-01 Подключение тарифа при достаточном балансе
**Дано:** Пользователь авторизован, баланс достаточен  
**Когда:** Выбирает тариф и нажимает «Подключить»  
**Тогда:** Тариф активирован, средства списаны

**Дано:** Пользователь авторизован, баланс недостаточен  
**Когда:** Выбирает тариф и нажимает «Подключить»  
**Тогда:** Отображается сообщение «Недостаточно средств», предложено пополнить баланс

### Сценарий: TS-02-03 Подключение уже активного тарифа
**Дано:** Пользователь имеет активный тариф  
**Когда:** Переходит к странице этого тарифа  
**Тогда:** Кнопка «Подключить» неактивна или заменена на «Управлять»

### Сценарий: TS-02-04 Подключение тарифа с балансом равным стоимости
**Дано:** Баланс равен стоимости тарифа  
**Когда:** Подключает тариф  
**Тогда:** Тариф активирован, баланс равен 0

## Группа 3: Домены

### Сценарий: TS-03-01 Регистрация домена с оплатой с баланса
**Дано:** Пользователь авторизован, баланс достаточен  
**Когда:** Вводит свободный домен, выбирает зону и оплачивает с баланса

**Дано:** Пользователь на форме поиска домена  
**Когда:** Вводит уже зарегистрированный домен  
**Тогда:** Отображается сообщение «Домен занят», кнопка регистрации неактивна

### Сценарий: TS-03-04 Регистрация при недостатке средств
**Дано:** Баланс недостаточен для регистрации домена  
**Когда:** Выбирает оплату с баланса
